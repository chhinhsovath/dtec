## How to Use Translation Keys

### In React Components
```typescript
import { useTranslation } from '@/lib/i18n/useTranslation';

export default function MyComponent() {
  const { t } = useTranslation();
  
  return <h1>{t('dashboard.student.title')}</h1>;
}
```

### Adding New Translation Keys

When implementing new features:

1. Add the key to `en.json`:
```json
{
  "newFeature": {
    "title": "New Feature Title",
    "description": "Description text"
  }
}
```

2. Add the corresponding translation to `km.json`:
```json
{
  "newFeature": {
    "title": "ចំណងជើងលក្ខណ៌ងថ្មី",
    "description": "អត្ថបទពិពណ៌នា"
  }
}
```

3. Use in component:
```typescript
const { t } = useTranslation();
return <h1>{t('newFeature.title')}</h1>;
```

## Best Practices

1. **Always use translation keys** - Don't hardcode text in components
2. **Use dot notation** - Organize keys hierarchically (e.g., `feature.section.item`)
3. **Keep keys short** - But descriptive enough to understand context
4. **Group related keys** - Organize by feature or page
5. **Test both languages** - Verify translations before deployment
6. **Document new keys** - Update this reference when adding new features
